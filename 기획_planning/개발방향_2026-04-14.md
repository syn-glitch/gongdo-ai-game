### 2.2 `ai_code_runner.html` 코드 레벨 검토

**현재 구조 (260줄)**
```
┌─ header (Run / Clear / Save 버튼)
├─ main-container
│   ├─ editor-container (textarea#code-input, 50% 폭)
│   └─ preview-container (iframe, sandbox)
└─ script (autosave + debounce 300ms + Cmd+Enter)
```

**강점**
- ✅ iframe `sandbox` 처리되어 안전
- ✅ localStorage 자동저장으로 작품 유실 방지
- ✅ 디바운스 300ms 프리뷰 — 성능 괜찮음
- ✅ 다크 테마, 단순·명료한 2-pane UI

**개발 시 교체·확장 포인트**
- ⚠️ 헤더 UI가 영어 위주 (Run/Clear/Save) → **한국어화** 및 초등 친화 아이콘 필요
- ⚠️ 플레이스홀더에 영문 예시 코드 → **한국어 안내 + 시작 템플릿 선택** 으로
- ⚠️ 좌/우 고정 50% 분할 → **챗봇 패널 추가**로 3-pane(또는 탭) 재구성 필요
- ⚠️ `doc.write()` 방식 프리뷰 → 안전하나 CSP 고려해 `srcdoc` 방식 검토
- ⚠️ 외부 통신 0 → Claude API 호출 로직 신설

## 4. 핵심 기술 결정

### 4.1 Claude API 챗봇 통합

**목적**: 뤼튼 대체 + 외부 가입 제거 + 교육용 제어 강화

**아키텍처**
```
학생 브라우저  ──프롬프트──▶  공도 서버(Edge Function)  ──시스템프롬프트+프롬프트──▶  Claude API
              ◀─HTML 패치─                            ◀─────────응답─────────
```

**서버 사이드 프록시가 필요한 이유**
- ❌ API 키를 브라우저에 두면 탈취 위험
- ✅ 서버에서 시스템 프롬프트·안전 필터·rate limit 적용
- ✅ 로그·비용 통제 가능

**기술 선택지**
| 옵션 | 장점 | 단점 |
| :--- | :--- | :--- |
| **Cloudflare Workers + Anthropic SDK** ⭐ 추천 | 무료 한도 넉넉, 저지연, 간단 | Workers 환경 학습 |
| Vercel Edge Functions | 배포 쉬움, Next.js 친숙 | 호출 수 한도 |
| Google Apps Script (GAS) | 기존 공도 스택 | 응답 지연·스트리밍 제약 |

**시스템 프롬프트 설계 방향** (초안)
```
당신은 초등 5~6학년에게 HTML 게임 제작을 돕는 친절한 AI 선생님입니다.
- 항상 한국어로 답하고, 초등학생이 이해할 수 있는 쉬운 단어를 씁니다
- 사용자의 요청에 따라 '완성된 단일 HTML 파일' 전체를 반환합니다
- 부적절한 내용(폭력·혐오·성적)은 정중히 거절합니다
- 코드 외 설명은 최소화하고, 핵심 변경점을 2~3문장으로 덧붙입니다
- 넷마블 캐릭터(ㅋㅋ, 토리, 밥, 레옹) 요청 시 제공된 이미지 URL을 사용합니다
```

**모델 선택**: `claude-haiku-4-5` (빠르고 저렴, 초등 수업 속도에 적합) 기본 + 복잡 요청은 `claude-sonnet-4-6` 폴백

**프롬프트 캐싱**: 시스템 프롬프트·샘플 게임 코드를 **prompt caching** 으로 고정 → 비용·지연 절감

### 4.2 🎵 인앱 음악 생성 (가입 없이, 서비스 내)

**대표 지시**: "음악이 다채롭지 않아도 되니 가입 없이 서비스 내에서 학생이 생성 가능해야 함"

#### 권장안 — **Tone.js + Claude 악보 생성** ⭐

**원리**
```
학생: "긴박한 우주 전투 음악"
   ↓
Claude (JSON 모드):
{
  "tempo": 140,
  "key": "C minor",
  "instrument": "synth-square",   // 8-bit 스타일
  "pattern": [
    {"note":"C4","dur":"8n"},
    {"note":"G4","dur":"8n"},
    ...
  ],
  "drums": ["kick","kick","snare","hihat", ...]
}
   ↓
브라우저 Tone.js: JSON 해석해 Web Audio로 실시간 재생
   ↓
MediaRecorder API: 학생이 원하면 .wav/.webm 파일로 내보내 게임에 삽입
```

**장점**
- ✅ **완전 가입 불필요** — 모든 합성이 브라우저에서 일어남
- ✅ **비용 0** — 서버는 Claude 호출만 담당 (JSON 생성, 토큰 작음)
- ✅ **즉각 반영** — 지연 1초 내
- ✅ **교육 효과** — 학생이 AI가 "악보를 쓴다"는 개념을 체감
- ✅ **게임 연동 자연스러움** — 8-bit 사운드가 레트로 게임 미학과 일치

**단점 & 완화**
- ❌ 보컬 불가능, 복잡 오케스트라 어려움 → 대표 지시대로 "다채로움 포기" 허용 범위 내
- ❌ 같은 프롬프트 → 비슷한 결과 → 🎲 seed 랜덤화로 변주 확보

#### 보조안 (폴백) — **SFX 프리셋 + BGM 무료 라이브러리**

- Web Audio 합성으로 **효과음**(레이저·점프·코인) 즉석 생성 — 파라미터 몇 개로 바리에이션
- Creative Commons 무료 BGM 10~20트랙을 교사 큐레이션으로 번들 → 학생이 AI 챗봇에 "밝은 분위기"라고 말하면 매칭해서 재생

### 4.3 넷마블 캐릭터 IP (v1.1 업데이트)

**현재 수령 자산** (`넷마블IP/`)

| 파일 | 형식 | 용량 | 웹 사용 가능 여부 |

| `넷마블프렌즈 메뉴얼 v2.0.ai` | Illustrator 원본 | 18 MB | ❌ 브라우저 직접 사용 불가 → ✅ 캐릭터 추출 원본으로 활용 |

**리사이즈 완료 자산** (`TRACK01_바이브코딩/assets/images/`)

| 파일 | 용량 | 용도 |
| :--- | ---: | :--- |

**가공 플랜 (최AR 담당)**
1. `.ai` 원본에서 **캐릭터별 투명 PNG/SVG** 추출 (ㅋㅋ·토리, 각 512×512 권장)
2. 표정·포즈 변형 3~4종(대기·점프·공격·승리) — 게임 캐릭터 교체 대응
3. `assets/characters/manifest.json` 에셋 맵 구성:
   ```json
   {
     "kk":    {"name":"ㅋㅋ",   "poses": {"idle":"...", "jump":"..."}},
     "tori":  {"name":"토리", "poses": {"idle":"...", "jump":"..."}}
   }
   ```
4. 브랜드 매뉴얼(`.ai` 내부)의 사용 규정(여백·색상·왜곡 금지 등)을 2페이지 요약 → 강사 배포

**챗봇 연동 방식**: 시스템 프롬프트에 `manifest.json` 내용을 캐싱으로 주입 → 학생이 "토리 캐릭터 점프하게 만들어줘" 하면 Claude가 해당 URL로 `<img>` 태그 자동 생성.

### 4.4 배포·인프라 (v1.1 확정)

**플랫폼**: **Vercel** (Hobby → Pro로 승급 권장, 상용 교육 서비스)
**도메인**: `ai-game.gongdo.kr` (서브도메인, 메인 도메인 `gongdo.kr` 보유 전제)
**임시 URL**: `gongdo-ai-game.vercel.app` (Vercel 기본, 도메인 연결 전 테스트용)
**화면 표기명**: "공도 AI-Game" (UI 타이틀)

```
┌──────────────────────────────────────────────┐
│  Vercel (정적 + Edge Function 통합)            │
│   ├─ / (정적: index.html, Tone.js, 에셋)       │
│   ├─ /api/chat   (Edge Function)              │
│   └─ /api/music  (Edge Function, 악보 JSON)    │
└──────────────────────────────────────────────┘

환경변수: ANTHROPIC_API_KEY  (Vercel Dashboard → Settings → Env)
접근 제어: Origin 헤더 검증 + rate limit (Upstash Redis 무료 티어)
SSL: Vercel 자동 발급 (Let's Encrypt)
```

**도메인 연결 절차 (요약)**:
1. Vercel Project → Settings → Domains → `ai-game.gongdo.kr` 추가
2. Vercel이 안내하는 CNAME 값(`cname.vercel-dns.com`)을
   gongdo.kr 도메인 관리 사이트 DNS 레코드에 추가
3. Vercel에서 "Verify" → SSL 자동 발급 → 완료

**비용**: 서브도메인 자체 무료, Vercel Hobby 무료(개인/테스트용), Pro $20/월(상용 운영 권장).

**학교 네트워크 고려**
- Google Fonts 등 CDN 차단 대비 → 폰트·라이브러리 로컬 번들
- HTTPS 필수 (sandbox iframe 요구사항)
- 오프라인 사전 로드 버전도 제공 (§3.3 P2)
