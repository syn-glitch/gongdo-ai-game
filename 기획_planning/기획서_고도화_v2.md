## 4. 커리큘럼 상세 — 5차시 타임라인

> **공통 구조**: 모든 차시는 **① Hook(5분) → ② Build(25분) → ③ Wrap(10분)** 의 3단계로 구성.

### 📖 1차시 — 바이브코딩 첫걸음

**학습목표**: AI에게 문장으로 말해 게임을 "만든다"는 것이 가능함을 체감한다.

| 시간 | 단계 | 활동 | 강사·플랫폼 |
| :---: | :--- | :--- | :--- |
| 0~5 | Hook | **샘플 게임 3종 시연** (점프 / 비행기 / 슈팅) — "우리도 오늘 하나 만들 거야" | 강사 시연, 샘플 게임 런처 |
| 5~10 | 개념 | 바이브코딩이란? (그림 카드 + 3분 영상) / 프롬프트 3요소 **[무엇을]·[어떻게]·[왜]** | 강사 설명, 워크북 p.1 |
| 10~30 | Build | 샘플 1개 선택 → 챗봇에 "배경을 ○○로 바꿔줘" 시도 → 결과 확인 → "적을 ○○로" 추가 | 공도 AI-Game 챗봇 |
| 30~38 | 저장 | 내 작품 제목 짓기 → **저장** 버튼 → 다운로드 확인 | 플랫폼 다운로드 |
| 38~40 | Wrap | 오늘 느낀 점 한 줄 (워크북) — "AI에게 한 내 말 중 가장 잘 통한 문장은?" | 워크북 p.2 |

**성취 확인**: 학생 HTML 파일 1개 저장 + 워크북 1줄 기록

---

### 📖 2차시 — AI로 넷마블 캐릭터 넣기

**학습목표**: IP 캐릭터를 게임에 삽입하고, 스토리·디자인 변수를 조작하는 경험.

| 시간 | 단계 | 활동 |
| :---: | :--- | :--- |
| 0~5 | Hook | 넷마블 프렌즈 소개 카드 — ㅋㅋ·토리·밥·레옹 이름과 성격 |
| 5~10 | 개념 | 캐릭터는 게임에서 **스프라이트** — 주인공을 바꾸면 분위기가 바뀐다 |
| 10~30 | Build | ① "주인공을 토리로 바꿔줘" ② "장애물을 주사위 모양으로" ③ "배경을 지구본맵으로" | 공도 AI-Game + 캐릭터 패널 |
| 30~38 | 저장 | 1차시 작품 이어서 저장 (v2) |
| 38~40 | Wrap | "어떤 캐릭터가 가장 우리 게임에 잘 어울렸나?" |

**사용 에셋**: `/assets/characters/*.png`, `/assets/images/worldmap_*.jpg`

---

### 📖 3차시 — AI로 BGM·효과음 만들기

**학습목표**: 소리가 게임에 미치는 영향을 체감하고, AI로 BGM을 생성·삽입한다.

| 시간 | 단계 | 활동 |
| :---: | :--- | :--- |
| 0~5 | Hook | 같은 게임, 다른 BGM 들려주기 — "느낌이 얼마나 다른지?" |
| 5~10 | 개념 | BGM은 **속도(BPM)** + **음색** + **분위기**로 설명할 수 있다 |
| 10~30 | Build | "긴박한 우주 전투 음악 만들어줘" → Tone.js 재생 → 마음에 들면 **추출** → 게임에 삽입 | 공도 AI-Game 음악 패널 |
| 30~38 | 저장 | 작품 v3 + BGM 포함 저장 |
| 38~40 | Wrap | "내 게임 BGM을 한 단어로 표현하면?" |

**기술 경로**: 학생 프롬프트 → Claude `/api/music` → JSON 악보 → Tone.js → MediaRecorder → .wav → 게임 `<audio>` 태그 자동 삽입

---

### 📖 4차시 — 발표 자료 만들기

**학습목표**: 내 작품을 남에게 설명하는 경험 + 자신이 쓴 프롬프트를 메타인지.

| 시간 | 단계 | 활동 |
| :---: | :--- | :--- |
| 0~5 | Hook | 어제 게임 개발자 발표 영상(30초) — "우리도 발표자가 된다" |
| 5~10 | 개념 | 발표 3요소: **이름·재미 포인트·만든 과정** |
| 10~30 | Build | ① 작품 제목·한줄소개 입력 ② 챗봇에 "이 게임 발표글 써줘" ③ 프롬프트 히스토리 자동 첨부 → 패들렛 업로드 | 공도 AI-Game 발표 모드 |
| 30~38 | 저장 | 패들렛 URL 확인 / 발표 자료(.md) 다운로드 |
| 38~40 | Wrap | 옆 친구 작품 하나 보고 "가장 멋진 점" 포스트잇 |

**산출물**: 작품 HTML + 프롬프트 히스토리 MD + 패들렛 포스트

---

### 📖 5차시 — AI 게임 공유회

**학습목표**: 다른 학생의 작품·프롬프트를 보며 배움을 확장 (peer learning).

| 시간 | 단계 | 활동 |
| :---: | :--- | :--- |
| 0~5 | Hook | "오늘은 발표회, 모두가 게임 디자이너" |
| 5~25 | Play | 반 전체 작품 순회 (패들렛 / 각자 화면 공유) — 3인 플레이·댓글 |
| 25~33 | 표창 | 카테고리별 "작은 상장" (가장 웃긴·가장 어려운·가장 특이한) |
| 33~38 | 사후설문 | 디지털 설문 (만족도·AI효능감·진로흥미) |
| 38~40 | Wrap | 한 줄 수료 소감 |

## 6. Claude 챗봇 시스템 프롬프트 설계 (v2 초안)

```
당신은 대한민국 초등 5~6학년 학생에게 HTML 게임 만들기를 도와주는
다정한 AI 선생님 "공도쌤"입니다.

【말투】
- 항상 쉬운 한국어로 답합니다. 한자어·IT 전문용어를 피합니다.
- 존댓말을 쓰되 친근하게 ("~해볼래요?" "좋은 생각이에요!").
- 초등학생이 모를 수 있는 단어는 괄호로 풀이를 답니다.

【코드 응답 규칙】
- 학생 요청에 따라 '완성된 단일 HTML 파일'을 <html>부터 </html>까지
  전체 반환합니다. 부분 수정이라도 전체 파일 형태로 응답합니다.
- 코드 블록은 반드시 ```html 로 감쌉니다.
- 코드 외 설명은 3문장 이내로, 변경한 핵심 포인트만 씁니다.

【안전】
- 폭력·혐오·성적·무섭거나 불쾌한 콘텐츠는 정중히 거절하고,
  대신 "귀여운 전투 게임"이나 "친구를 돕는 게임" 같은 대안을 제시합니다.
- 개인정보(이름·학교·주소)를 게임에 넣자고 하면 익명 캐릭터 사용을 권합니다.

【넷마블 캐릭터】
- 아래 캐릭터 에셋 맵을 활용해 이미지 태그를 자동 삽입합니다:
  {MANIFEST_JSON}
- 학생이 캐릭터 이름을 부르면 해당 URL을 <img> 로 게임에 넣어줍니다.

【게임 규모 제한】
- 학생이 40분 수업 중이므로 코드는 300줄 이내로 유지합니다.
- 외부 스크립트(CDN)는 허용 목록 외 사용하지 않습니다:
  허용: Google Fonts, Tone.js CDN
```

**캐싱**: 위 시스템 프롬프트 + 샘플 게임 3종 코드를 **Anthropic Prompt Caching**으로 고정 → 입력 토큰 비용 ~90% 절감.

  ],
  "mood": "긴박한 우주 전투",
  "instruments": {
    "melody": "synth-square",
    "bass":   "synth-triangle",
    "drums":  "noise"
  }
}
```

**처리 흐름**: `/api/music` → Claude에게 위 스키마로 JSON 반환 요청 → Tone.js 인터프리터가 해석해 Part·Sequence 스케줄 → Transport.start() → 사용자가 마음에 들면 MediaRecorder로 .wav 추출 → 게임에 `<audio>` 자동 삽입.

**제약**: 8-bit 스타일 고정 (synth-square/triangle/noise), 보컬 없음. → 대표 승인된 "다채로움 포기" 허용 범위 내.
