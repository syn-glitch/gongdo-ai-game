### 🚨 발견 이슈 3건

#### MAJOR-1 — Fallback 시 이전 하이라이트 잔존
- **재현**: 주인공 클릭(22 라인 하이라이트) → 장르 클릭(매칭 실패)
- **기대**: 매칭 실패 시 하이라이트 클리어 + 토스트 안내
- **실제**:
  · 토스트: ✅ "🔎 '장르' 부분과 딱 맞는 코드를 못 찾았어요..." (정상)
  · 화면: ❌ **이전 22 라인 하이라이트 그대로 표시됨**
- **근거 데이터**: AC-3 검증 시 `hlAfterFallback=11` (이전 "조작 방법" 11 라인 잔존)
- **영향**: 학생이 "내가 클릭한 게 작동 안 하나?" 혼란 OR
       교사가 "장르 = 이 코드" 라고 잘못 가르치는 위험
- **원인 추정 (자비스 위임)**:
  ```js
  // app.js highlightCodeSection() Fallback 분기
  if (!range) {
    $('#game-status').textContent = `🔎 ... 못 찾았어요...`;
    return;  // ← 여기서 return 만 함, 이전 하이라이트 클리어 X
  }
  ```
  → return 전 `showFullCode()` 호출 추가 권장
- **등급**: MAJOR (학생 혼란 + 학습 효과 저해)

- **분석**: AI 학생용 주석에 "장르" 자체를 인용하는 경우 거의 없음
  ("장르 = 우주 슈팅 게임" 은 게임 자체의 메타 정보, 특정 함수에 매핑 X)

  - B. 매칭 실패가 정상이라 보고 그대로 (Fallback 안내가 학습)
  - 송PO 결정 권장
- **등급**: MINOR (사용성, 학습 효과 직접 저해 X)
