## 발견 이슈 (Problem)

### 🚨 MAJOR-1 — AC 미커버 FR 4건 (완전성 결함)
| FR | 내용 | 누락 AC |
|----|------|---------|
| FR-2 | 버튼 라벨 "↻ 처음으로" | 라벨 정확 검증 AC 없음 |
| FR-4 | 인라인 popover (modal X) | AC-FR-8 에 일부 있으나 "인라인" 구분 모호 |
| FR-6 | confirm 버튼 라벨 정확 ([✓ 네, 처음으로] / [✗ 아니에요]) | 라벨 정확 검증 AC 없음 |
| FR-11 | "아니오" 시 popover 닫기만 | AC-UX-2 가 editor 만 검증, popover 닫힘은 미검증 |
**영향**: 자비스가 라벨을 다르게 구현해도 AC 미통과로 잡지 못함

### 🚨 MAJOR-2 — 엣지 케이스 누락: 동시 popover
- 시나리오: 학생이 [캐릭터] popover 가 열린 상태에서 [↻ 처음으로] 클릭 시
- 명세: 동작 미정의
- **권장 추가 AC-Edge-1**: 다른 popover (캐릭터/배경/예시/음악) 가 열린 상태에서
  [↻ 처음으로] 클릭 시 → 기존 popover 닫고 confirm popover 노출

### 🚨 MAJOR-3 — 엣지 케이스 누락: confirm 중 다른 액션
- 시나리오 A: confirm popover 노출 중 학생이 차시 트리에서 **다른 차시 클릭**
- 시나리오 B: confirm 노출 중 학생이 **[시작]** 클릭
- 시나리오 C: confirm 노출 중 학생이 **[↻]** 다시 클릭 (또는 [캐릭터] 등)
- 명세: 미정의
- **권장 추가 AC-Edge-2/3/4**:
  · A → confirm 자동 닫힘 + 차시 전환 정상
  · B → confirm 자동 닫힘 + [시작] 동작 정상 (또는 confirm 우선)
  · C → 토글 (다시 누르면 confirm 닫힘)

### 🚨 MAJOR-4 — 엣지 케이스 누락: cache miss 시 동작
- 명세 §6.3 에 실패 토스트 문구만 있음 ("⚠️ 처음 예시를 불러오지 못했어요")
- §8 AC 에 cache miss 검증 없음
- FR-7 은 "lessonCache.get(현재 file)" 만 명시 — cache 가 없으면 어떻게?
  · 옵션 A: 즉시 fetch (NFR-3 100ms 위반 가능)
  · 옵션 B: 토스트만 + editor 변경 X (가장 안전)
- **권장 추가 FR-16**: cache miss 시 자동 fetch + 실패 시 토스트
- **권장 추가 AC-Edge-5**: cache 비운 후 [↻] 클릭 → fetch 재시도 OR 친절한 실패 안내

### ⚠️ MINOR-1 — 모호: 5초 timeout 중 hover 처리
- §FR-13: "5초 미응답 시 자동 닫힘"
- 사용자가 popover 안에 마우스 hover 중일 때 timer 진행되는가? 일반 UX 는 hover 중 reset
- **권장**: FR-13 에 "단, popover 내 마우스 hover 중에는 timer 일시정지" 명시 OR "단순히 5초 절대 시간" 명시

### ⚠️ MINOR-2 — 모호: NFR-3 100ms 측정 방법
- "버튼 클릭 → editor 갱신까지 100ms 이내"
- 측정 도구: console.time? Performance API? 시각 인지?
- **권장**: AC-Perf-1 에 측정 방법 명시 (예: `performance.now()` 차이 ≤ 100ms)

### ⚠️ MINOR-3 — 모호: FR-5 본문 \n 의미
- "정말 처음 예시로 되돌릴까요?\n지금까지 바꾼 내용은 사라져요!"
- \n 이 줄바꿈인지 별도 div 인지 불명확
- **권장**: HTML 구조 명시 (예: 두 개의 `<p>` 또는 `<br>`)

### ⚠️ MINOR-4 — 누락: 1차시 외 차시에서 [예시] 버튼 동시 동작
- 1차시는 [예시] 토글 + [↻ 처음으로] 둘 다 노출
- 학생이 [예시] 로 변형 선택 → 수정 → [↻] 누르면 어느 원본?
- AC-회귀-5 에 "variants 선택 직후 초기화 → 해당 variant 의 원본" 명시 있음 ✅
- 그러나 "[예시]로 변형 변경 → 잠시 수정 → [↻]" 의 시퀀스는 명시 X
- **권장**: AC-회귀-5 를 "[예시] 변경 후 수정 → [↻] 시 마지막 선택 variant 원본" 으로 강화

### ⚠️ MINOR-5 — 누락: 학생이 토스트 못 보는 경우
- 토스트는 game-status 영역 1줄 표시
- 학생이 게임 화면을 보고 있어 game-status (상단) 미인지 가능
- 김감사 QA-002 에서 동일 패턴 지적했던 이슈 (인지 불일치)
- **권장**: 초기화 후 편집기에 짧은 시각 효과 (flashEditor() 재활용) 추가 검토
