## 이슈 분류

### ⚠️ MAJOR — UX 인지 불일치
- **증상**: 캐릭터 변경 시 편집기·토스트는 즉각 반영되나, 게임 iframe 화면은 [시작] 전까지 이전 상태 유지
- **영향**: 학생이 "캐릭터 변경 = 즉시 게임 반영" 으로 기대 → 실제 미반영 → 버그로 오인 → 신뢰 저하
- **재현율**: 100% (모든 학생이 동일 동선에서 동일 오인 가능성)

### 📝 MINOR — 토스트 시각 강조 부족
- **증상**: "[시작]을 눌러봐요!" 메시지가 우측 상단 작은 글씨로만 표시
- **현상**: 게임 화면을 보고 있는 학생의 시선이 토스트로 향하지 않음
- **개선 여지**: 게임 영역 자체에 변화 안내가 없음

## 권장 조치 (Recommended Actions)

### 옵션 A — 자동 재생성 (가장 직관적, 비용 발생)
- 캐릭터/배경/배경음악 변경 시 자동으로 [시작] 트리거
- 장점: 학생 인지 일치 100%, 동선 단축
- 단점: rate limit (5회/일) 빠르게 소진, 토큰 비용 증가
- 권고: 차시별 게임 생성 한도 검토 필요 (현재 5회 → 7~10회?)

### 옵션 B — 게임 화면 위 안내 오버레이 (균형)
- 캐릭터/배경 변경 시 게임 iframe 위에 반투명 오버레이:
  ```
  ⚠️ 캐릭터를 '토리'로 바꿨어요!
       [▶ 시작]을 눌러야 게임에 반영돼요
       [지금 바로 시작]  ← 큰 버튼
  ```
- 장점: 비용 0, 학생 인지 100%, 자율적 [시작] 클릭 유도
- 단점: 게임 진행 방해 (게임 중 캐릭터 변경 시)

### 옵션 C — [시작] 버튼 펄스 + 뱃지 (가장 가벼움)
- 캐릭터 변경 시 [시작] 버튼에:
  - 펄스 애니메이션 (3초간)
  - 작은 빨간 점 뱃지 ("변경사항 있음")
- 장점: 비용 0, 게임 진행 방해 없음
- 단점: 토스트 못 본 사용자는 여전히 못 알아챌 가능성

### 추천 조합: B + C
- 일반 캐릭터 변경 → C (가벼운 펄스 + 뱃지)
- 게임 일시정지 상태에서 변경 → B (오버레이 + "지금 시작" 버튼)
