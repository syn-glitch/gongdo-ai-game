## 발견 이슈 7건

### 🚨 MAJOR-1 — NFR-5 AC 누락 (한국어 어휘)
- NFR-5 "영역 헤더·설명 모두 초등 5~6 친화" → §8 AC 어디에도 검증 없음
- BUNKER-003 의 AC-NFR-6 (UI 어휘 IT 용어 0건) 와 동일 패턴이나 본 PRD 에서 빠짐
- **권장 추가 AC-NFR-5**: 영역 헤더 ("📝 내 바이브코딩 문서", "내가 한국어로 쓴 거예요" 등) 에 IT 용어 0건
       grep 검증: `grep -E "변수|함수|조건문|루프|메서드|클래스|인스턴스|콜백" index.html` 결과 0

### 🚨 MAJOR-2 — XSS / HTML 특수문자 escape 검증 부재
- 학생이 마크다운에 `<script>alert(1)</script>` 또는 `<img onerror=...>` 입력 가능
- §9.2.1 renderMarkdownLite 가 escapeHtml 호출하나 AC 에 검증 없음
- **권장 추가 AC-Edge-6**: 마크다운에 `<script>` / `<img>` / `&` / `"` 입력 시
       마크다운 영역에 **escape 된 텍스트로 표시** + script 실행 0건
       (Console: `confirm` 호출 없음 + innerHTML 에 `&lt;script&gt;` 형태)
- **이유**: 학생이 호기심으로 입력 가능 + 다른 학생 화면 공유 시나리오에서 실제 위험

### 🚨 MAJOR-3 — FR-12 동기성 vs FR-13 fallback 잠재 충돌
- FR-12: 마크다운 영역과 코드 영역이 항상 같은 시점 (snapshot 동기성)
- FR-13: snapshot 없는 구버전 게임 → 마크다운 영역은 `editor.value` fallback
- **충돌**: 학생이 구버전 게임을 본 뒤 편집기를 수정하면 → 마크다운(현재값) ≠ 코드(옛 게임) → FR-12 위배
- **권장 선택**:
  - 옵션 A: FR-13 fallback 제거 — 구버전 게임은 마크다운 영역에 안내 ("옛날 게임이라 문서를 표시할 수 없어요. 새로 [▶ 시작] 눌러봐요!")
  - 옵션 B: FR-13 유지하되 노란 안내 배지 추가 ("옛날 게임이에요 — 마크다운은 현재 편집기 내용이에요")
- **송PO 결정 필요** — 옵션 A 권장 (FR-12 학습 효과 우선)

### 🚨 MAJOR-4 — 측정법 모호 (AC-FR-3b, AC-FR-12)
- AC-FR-3b: "한쪽 스크롤이 다른 영역 영향 0" → 어떻게 측정?
- AC-FR-12: "snapshot 동기성" → 측정 절차 모호
- **권장 보강**:

  - AC-FR-12: `state.lastGeneratedHtmlSnapshot.sourceText` 와 마크다운 영역 innerText 비교
       (escape 차이 감안)

### ⚠️ MINOR-1 — 마크다운 inline 스타일링 처리 검증 누락

### ⚠️ MINOR-2 — CSS 마이그레이션 방침 모호
