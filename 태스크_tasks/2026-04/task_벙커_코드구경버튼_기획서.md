## 1. 지시 원문
> 바이브코딩 문서가 현재는 마크다운 한글인데, 코드 보기 버튼 추가:
> - 초등학생에게 친숙한 용어의 버튼
> - 클릭 시 바이브코딩 문서와 일치하는 코드 노출
> - 코드 단락마다 마크다운 문서가 어떻게 되어 있는지 주석 설명
> - 간단한 코드 설명 (초등학생도 이해)
> - 목적: 코드 언어 ↔ 자연어 매핑 학습
> - 코드 화면은 뷰어 전용 (read-only)
> - 문서 변경 후 다시 [코드 보기] 누르면 갱신

## 2. 배경 및 목적

### 2.1 배경
- 학생은 한국어 마크다운으로 게임을 "지시"하지만, 실제 게임이 어떻게 만들어지는지 코드 측면 노출 0
- "내가 쓴 문장이 어떻게 진짜 코드가 되는지" 호기심·학습 유발 미흡
- AI 가 만드는 결과물(코드)과 학생이 만드는 입력물(자연어) 사이 **블랙박스** 존재

### 2.2 목적
- **언플러그드 → 플러그드 학습 전환의 다리**
- 학생이 코드를 두려워하지 않게 (코드는 그저 "조립된 영어 단어들" 임을 인지)
- 자연어 → 코드 1:1 매핑 시각화 → 컴퓨팅 사고력 학습
- 향후 "코드도 직접 고쳐볼래요?" 단계로 확장 가능 (이번 PR 외)

### 2.3 비목적 (Out of Scope)
- 학생이 코드 직접 수정 (이번은 read-only 만)
- 코드 문법 자동 검사·실행
- 4·5차시 적용 (게임 생성 의미 적음 → hide)
- 발전형 E3 (코드 클릭 시 마크다운 하이라이트) — 별도 task

## 3. 사용자 시나리오

### 3.1 행복 경로 (Happy Path)
1. 학생이 1차시 → 우주 슈팅 → [▶ 시작] → 게임 생성
2. 학생이 도구바 [🔍 코드 구경] 클릭
3. 편집기 위에 코드 오버레이 노출
4. 헤더: "👀 보기만 해요! 바꾸려면 ✕ 닫고 문서를 고쳐요"
5. 본문: 실제 게임 HTML/JS 코드 + 학생용 주석:
   ```js
   // 📝 "주인공: ㅋㅋ" → 너의 주인공 ㅋㅋ 그리는 부분이야!
   function drawPlayer(ctx, x, y) { ... }
   ```
6. 학생이 코드 구경 후 ✕ 닫기 (또는 Escape)
7. 편집기 그대로 유지

### 3.2 게임 미생성 (Pre-Game)
1. 학생이 차시 열기만 한 상태 ([시작] 미클릭)
2. [🔍 코드 구경] 클릭
3. 오버레이 안내: "🎮 먼저 [▶ 시작]을 눌러 게임을 만들어요!\n그러면 코드를 구경할 수 있어요."

### 3.3 문서 변경 후 (Stale)
1. 학생이 게임 생성 → 코드 구경 한 번
2. 닫고 편집기에서 "주인공: ㅋㅋ" → "주인공: 토리" 수정 ([시작] 미클릭)
3. [🔍 코드 구경] 다시 클릭
4. 오버레이 본문 위에 안내 배지: "📝 문서를 바꿨어요! [▶ 시작]을 다시 눌러야 새 코드가 보여요"
5. 그 아래는 이전 게임 코드 그대로 (혼란 방지)

### 3.4 4·5차시 (적용 외)
- [🔍 코드 구경] 버튼 자체가 hide

### 9.2 구현 힌트 (송PO 권고, 강제 X)
- 오버레이는 `.pane-editor` 의 자식, `position: absolute; inset: 0` + `z-index` 로 덮음
- `<pre><code id="code-view-content">` 에 `state.lastGeneratedHtml` 텍스트로 삽입 (innerText 사용 → XSS 방지)
- syntax highlighting 은 정규식 기반 CSS span 래핑:
  ```js
  function tokenize(code) {
    return code
      .replace(/(\/\/[^\n]*)/g, '<span class="code-tag-comment">$1</span>')
      .replace(/(&lt;!--[\s\S]*?--&gt;)/g, '<span class="code-tag-comment">$1</span>')
      .replace(/('[^']*'|"[^"]*")/g, '<span class="code-tag-string">$1</span>')
      .replace(/\b(function|const|let|var|if|else|return|new)\b/g, '<span class="code-tag-keyword">$1</span>');
  }
  ```
  주의: HTML escape 먼저 (`&` → `&amp;`, `<` → `&lt;`)

- **색상 토큰 (v1.1, 송PO 결정 — 자비스 자율 결정 X):**
  | 클래스 | 색상 | 추가 스타일 |
  |--------|------|------------|
  | `.code-tag-comment` | `var(--color-brown-soft)` | `font-style: italic` |
  | `.code-tag-keyword` | `var(--color-brown)` | `font-weight: bold` |
  | `.code-tag-string`  | `var(--color-green, #2d5f2d)` | — |
  | (나머지) | 기본 본문 색 | — |
  → 디자인 토큰 일관성 + comment ↔ keyword 명확 구별
- 변경 감지: state.lastGeneratedHtmlSnapshot.sourceText 와 editor.value !== 비교
- chat.js SYSTEM_GENERATOR 에 새 블록 추가:
  ```

  코드 안에 // 📝 "[학생 문서 인용]" → [초등 5~6 친화 한 줄] 형식 주석

  ```
